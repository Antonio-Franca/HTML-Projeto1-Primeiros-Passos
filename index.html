Comunicação Cliente X Servidor

Voltar ao índice

Antes de nos aventurarmos no mundo das páginas web, precisamos compreender um conceito muito importante: como se dá a comunicação na internet. Para isso, precisamos saber a arquitetura ou relação cliente e servidor.A relação cliente-servidor é um conceito fundamental em computação e redes, o qual descreve a interação entre dois tipos de programas ou dispositivos: o cliente, que faz solicitações de serviços ou recursos, e o servidor, que fornece esses serviços ou recursos em resposta às solicitações do cliente.

Cliente

O cliente é geralmente um programa ou dispositivo que solicita serviços ou recursos de um servidor. Ele envia uma solicitação ao servidor especificando qual serviço ou recurso ele precisa e aguarda uma resposta do servidor. Os clientes podem variar em complexidade, desde um navegador da web solicitando uma página da web até um aplicativo de email solicitando o envio de uma mensagem. Entre suas características temos:

  1. Inicia as Solicitações: O cliente é o iniciador da comunicação. Ele envia solicitações para o servidor solicitando serviços ou recursos específicos.
  2. Interage com o Usuário: Em muitos casos, o cliente interage diretamente com o usuário final. Por exemplo, um navegador da web, cliente de email ou aplicativo de mídia social.
  3. Recebe respostas: O cliente espera receber respostas do servidor em resposta às suas solicitações. Essas respostas podem incluir dados, recursos ou confirmações de que a solicitação foi processada com sucesso.

Servidor

O servidor é um programa ou dispositivo que recebe as solicitações dos clientes, processa essas solicitações e envia de volta as respostas apropriadas. Os servidores são projetados para fornecer serviços específicos, como hospedagem de sites, armazenamento de arquivos, processamento de emails, entre outros. Eles são configurados para escutar solicitações de clientes em uma porta específica e, quando uma solicitação é recebida, o servidor a processa e envia a resposta de volta ao cliente.Entre suas características temos:

  1. Aguarda Solicitações: O servidor permanece em um estado passivo, aguardando solicitações de clientes. Ele responde a essas solicitações, processando-as e fornecendo as respostas apropriadas.
  2. Alta Capacidade de Processamento e Armazenamento: Os servidores geralmente são equipados com alta capacidade de processamento, memória e armazenamento para lidar com múltiplas solicitações de clientes simultaneamente.
  3. Escalabilidade: servidores são projetados para serem escaláveis, o que significa que podem ser dimensionados para lidar com um grande número de clientes simultaneamente. Isso pode ser alcançado por meio de técnicas como balanceamento de carga e clustering.

Como se dá a comunicação

A comunicação entre clientes e servidores geralmente ocorre através de redes, como a internet. As solicitações são enviadas pelos clientes para os endereços IP dos servidores, geralmente especificados por meio de URLs ou outros identificadores de rede. As respostas do servidor são enviadas de volta para o cliente, completando o ciclo de comunicação.

O que é HTML?

Voltar ao índice

HTML(HyperText Markup Language ou Linguagem de Marcação de Hipertexto)é a linguagem padrão utilizada para criar e projetar páginas da web. Ele fornece a estrutura básica para páginas da web como o layout, o conteúdo e os elementos interativos de uma página. O HTML usa marcação para definir elementos individuais dentro de uma página, como: cabeçalhos, parágrafos, imagens, links e muito mais. Esses marcadores são então interpretados pelos navegadores da web para exibir o conteúdo da página conforme projetado. Diferentemente de uma linguagem de programação o HTML é uma linguagem de marcação, enquanto linguagens de programação como Python, JavaScript ou Java são linguagens de programação de propósito geral. Em HTML, você utiliza tags para marcar partes do conteúdo, indicando sua função e estrutura, enquanto em linguagens de programação você escreve instruções que são executadas sequencialmente.

O que são Tags?

Voltar ao índice

Antes de falarmos da estrutura básica de um documento HTML, Precisamos falar de tags: o coração do HTML. As tags são elementos básicos que são usados para marcar e definir diferentes partes do conteúdo de uma página da web. As tags são cercadas por colchetes ("<" e ">") e geralmente aparecem em pares, com uma tag de abertura e uma tag de fechamento, embora algumas tags possam ser autocontidas.

Estrutura Básica de um documento HTML

Voltar ao índice

Em relação a estrutura básica de um documento HTML, pode-se dizer que consiste nas seguintes tags:

  • <!DOCTYPE html>: Esta declaração define o tipo de documento e a versão do HTML que está sendo usada.
  • <html>: A tag html é o contêiner raiz de todo o documento HTML. Ele engloba todo o conteúdo da página.
  • <head>: A tag head contém metadados sobre o documento, como o título da página, links para folhas de estilo externas (CSS), scripts JavaScript, codificação de caracteres e outras informações que não são diretamente renderizadas na página.
  • <body>: A tag body contém todo o conteúdo visível da página, como texto, imagens, vídeos, links e outros elementos HTML.

Principais tags

Voltar ao índice

Aqui estão algumas das tags HTML mais comuns:

  • <title>: Define o título da página, exibido na barra de título do navegador.
  • <h1> a <h6>: São os cabeçalhos, onde <h1> é o mais importante e <h6> o menos importante.
  • <p>: Define um parágrafo de texto.
  • <a>: Cria um hiperlink para outra página ou recurso.
  • <img>: Insere uma imagem na página.
  • <ul> e <ol>: Criam listas não ordenadas (com marcadores) e ordenadas (numeradas), respectivamente.
  • <li>: Define um item de lista dentro de <ul> ou <ol>.
  • <div> e <span>: São elementos genéricos usados para agrupar outros elementos e aplicar estilos.
  • <table>: Define uma tabela.
  • <tr>: Define uma linha em uma tabela.
  • <td>: Define uma célula de dados em uma tabela.
  • <form>: Define um formulário para entrada de dados do usuário.
  • <input>: Cria um campo de entrada em um formulário.
  • <button>: Cria um botão clicável.
  • <textarea>: Cria uma área de texto multi-linha em um formulário.
  • <label>: Define uma etiqueta para um elemento de formulário.

Essas são apenas algumas das tags HTML, mas existem muitas outras para várias finalidades e funcionalidades específicas.
